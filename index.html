AccuTransitSim Overview

Key Features

AccuTransitSim is a random location coordinate generator with real-time speed adjustments. Buses can move on a custom-generated map with fewer coordinates. The GPS tracker updates the coordinates at one-second intervals. This app accurately generates the bus's location based on its speed (not purely random).

  • Generates the route by matching the given coordinates.
  • Calculates the bus’s real-time location in the route based on distance per second.
  • Calculates the distance to the next bus stop on every coordinate generation.
  • Simulates buses moving forward or backward along each route.

Controller & Dashboard

The simulator has a Dashboard and Controller with two main tabs:

Controller (Tab 1)

  • Route Direction – Forward or reverse of each route
  • Lowest & Highest Speed – Bus speed changes between these two values
  • Starting Time – After a given time, the bus will start transit
  • Add Button – Add multiple buses to the list for simultaneous simulation
  • Simulate Button – Start the simulation of all buses in the list

Speed Adjusting (Tab 2)

  • Bus Number – Add the bus number to load its current lowest and highest speeds
  • Upper & Lower Arrows – Increase or decrease the lowest and highest speeds dynamically

Dashboard

  • Map of Routes – XY coordinate pane showing all routes
  • Distance to Next Bus Stop – Displayed for each bus
  • Bus States – Scheduled start time, in-transit state, destination arrival time, total travel time, departure time

Logical Background

The AccuTransitSim application consists of three key projects:

  1. Bus & GPS Simulator with Controller
  2. Transit Operation Centre
  3. Dashboard with Map

Bus & GPS Simulator with Controller
This component reads JSON files containing route IDs, route coordinates, and bus stop coordinates. It simulates real-time GPS tracking, using the bus’s speed and angle to determine the next coordinate. It also calculates distance to the next bus stop at every step. Updates are broadcast at 1 Hz frequency.

Transit Operation Centre
Collects data from the GPS Tracking Unit and also reads map files. The Dashboard fetches data from the Transit Operation Centre, rather than directly from the GPS simulator.

Dashboard with Map
Displays the bus’s routes and states in near-real time (at 0.8 Hz).

Used Technologies

  • Bus & GPS Simulator with Controller – Spring Boot & JavaFX
  • Threads for each active GPS tracking unit
  • Route data stored in an ArrayDeque, used with Guava’s Peeking Iterator
  • Transit Operation Centre – Spring Boot / REST API
  • Dashboard with Map – React.js / Canvas API
